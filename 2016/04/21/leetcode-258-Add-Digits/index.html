leetcode-258-Add-Digits 解题及思考

  • LeetCode

这是我刷的第二道leetcodeOJ,在此题中用到了ArrayList。上干货：

题目

Given a non-negative integer num, repeatedly add all its digits until the result has only one digit.

For example:

Given num = 38, the process is like: 3 + 8 = 11, 1 + 1 = 2. Since 2 has only one digit, return it.

解

分析

在刷leetcode时发现，leetcode会直接给出定义的class和方法名，这其实给出了一定的提示，起码告诉你返回值是什么类型。但值得注意的，其未给出主方法，但程序仍可运行，这说明，其已经封装好了。并且可以定义新的方法并调用，在此题中便定义了新方法GetArr(intnum)，并调用，但为了避免出错，将其定义为static，再刷新题时，试下不加static看能否直接调用。

解此题，关键是将输入的数字按位分割，并将各位上的数字存储，然后将各位上的数字相加，开始想到过将数字转换为字符串，再将字符串分割成字符，再将字符转换为数字相加，很麻烦，并且字符串分割成字符时用到split()方法，需要指定分隔符，而字符串是连续的数字时没有分隔符，这如何分割？思考下这个问题，思考到这便遇到了瓶颈，于是想其他方法。

想到了数组，对数字进行除以10再取余数便可分割数字的每一位，再用数组存储；但是用数组有个问题，声明数组时，必须指定数组的类型和长度，因为需要为其分配内存空间，而我们并不知道输入的数字是多少位的，于是无法确定数组的大小。所以用数组也不行，这时我想，有没有一种数据存储结构可以像数组一样按下标存储并且是变长的？我都不知道java里也有list，我是个java渣渣，楠哥告诉我可以用list。果然问题解决了。
