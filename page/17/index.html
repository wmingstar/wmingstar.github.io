leetcode-258-Add-Digits 解题及思考

这是我刷的第二道leetcodeOJ,在此题中用到了ArrayList。上干货：

题目

Given a non-negative integer num, repeatedly add all its digits until the result has only one digit.

For example:

Given num = 38, the process is like: 3 + 8 = 11, 1 + 1 = 2. Since 2 has only one digit, return it.

解

分析

在刷leetcode时发现，leetcode会直接给出定义的class和方法名，这其实给出了一定的提示，起码告诉你返回值是什么类型。但值得注意的，其未给出主方法，但程序仍可运行，这说明，其已经封装好了。并且可以定义新的方法并调用，在此题中便定义了新方法GetArr(intnum)，并调用，但为了避免出错，将其定义为static，再刷新题时，试下不加static看能否直接调用。

解此题，关键是将输入的数字按位分割，并将各位上的数字存储，然后将各位上的数字相加，开始想到过将数字转换为字符串，再将字符串分割成字符，再将字符转换为数字相加，很麻烦，并且字符串分割成字符时用到split()方法，需要指定分隔符，而字符串是连续的数字时没有分隔符，这如何分割？思考下这个问题，思考到这便遇到了瓶颈，于是想其他方法。

想到了数组，对数字进行除以10再取余数便可分割数字的每一位，再用数组存储；但是用数组有个问题，声明数组时，必须指定数组的类型和长度，因为需要为其分配内存空间，而我们并不知道输入的数字是多少位的，于是无法确定数组的大小。所以用数组也不行，这时我想，有没有一种数据存储结构可以像数组一样按下标存储并且是变长的？我都不知道java里也有list，我是个java渣渣，楠哥告诉我可以用list。果然问题解决了。

分布式数据库与并行数据库

分布式数据库概述

概念

分布式数据库系统是物理上分散（数据分布在集群中的不同节点）、逻辑上集中的数据库系统，通信网络将集群中的不同节点连接起来，每个节点既有独立处理的能力，也可以和其他场地协同工作。

分布式数据库是分布式数据库系统中各节点上数据库的逻辑集合。
分布式数据库系统中有两个重要的组成部分:分布式数据库(DDB)和分布式数据库管理系统(DDBMS)。

分布式数据库的目标

1987年，C,J.Date提出了分布式数据库要达到的12各目标：
（1）本地自治；（2）非集中式管理；（3）高可用性；（4）位置独立性；（5）数据分片独立性；（6）数据复制独立性；（7）分布式查询处理；（8）分布式事务管理；（9）硬件独立性；（10）操作系统独立性；（11）网络独立性；（12）数据库管理系统独立性；

本地自治、非集中式管理、高可用性是分布式数据库最基本的特征；
位置独立性、数据分片独立性、数据复制独立性形成了分布式数据库系统中的分布透明性，使用户感觉不到数据是分布的，与使用集中式数据库完全一样；
分布式查询处理与分布式事务管理给分布式数据库系统带来了一定的复杂性。

数据分布策略

数据分片按照一定规则将一个全局关系划分为片断；数据分配则在此基础上将这些片断分配存储在各个节点上。分片是对关系的操作，而分配则是对分片结果的操作。

数据分片

对某一个关系进行分片是将关系划分成多个片断，这些片断中包含足够的信息可以使关系重构；数据分片有四种基本方法：A.水平分片；B.垂直分片； C.导出分片； D.混合分片；

数据分配

数据分配有以下几种方法：A.集中式；B.分割式； C.全复制式； D.混合式；

分布式数据库的特点

（1）数据具有物理分布性。即数据库中的数据不是集中存储在一个地区的一台计算机上，而是分布在不同场地的计算机上。
（2）数据逻辑整体性。虽然数据库物理上是分布的，但在逻辑上却是一个相互联系的整体。
（3） 数据具有分布独立性。
（4）场地自治与协调。系统中的每个节点都具有独立性，能执行局部应用，同时每个节点又是整个系统的一部分，可通过网络执行全局应用。
（5） 数据具有一定的冗余，冗余是透明性。
四个基本特点:
(1) 物理分布性(2) 逻辑整体性(3) 节点自治性(4) 节点之间协作性
四个导出特点:
(5)数据独立性(6) 集中与自治相结合的控制机制(7) 适当增加数据冗余度 (8) 事务管理的分布性

分布式数据库的优缺点

优点:

(1)具有灵活的体系结构。(2)适应分布式的管理和控制机构。(3)经济性能优越。(4)系统的可靠性高、可用性好。(5)局部应用的响应速度快。(6)系统可扩展性好。

缺点

(1)系统开销较大，主要花在通信部分。
(2)存取结构复杂(如辅助索引、文件链接技术)。
(3)数据的安全性和保密性较难处理。

并行数据库

概念

并行处理技术是利用多个处理器来共同完成一项任务，是计算机技术的一个重要发展方向。
并行数据库是数据库技术与并行处理技术相结合的产物。
并行数据库系统以高性能(线性加速比)、高可用性与高扩充性(线性伸缩比)为目标，充分利用多处理器平台的能力，通过多种并行性，在联机事务处理与决策支持应用两种典型环境中提供优化的响应时间与事务吞吐量。
体系结构：紧耦合全对称多处理器(SMP)系统；松耦合群集机系统；大规模并行处理(MPP)系统；混合结构。

分布式数据库与并行数据库的不同

应用目标不同

并行数据库系统的目标是充分发挥并行计算机的优势，利用系统中的各个节点并行完成数据库任务，提高数据库系统的整体性能。分布式数据库系统主要目的在于实现节点自治和数据的全局透明共享，而不要求利用网络中的各个结点来提高系统处理性能。

实现方式不同

并行数据库系统为了充分利用各个节点的处理能力，各节点间可以采用高速网络连接。节点间的数据传输代价相对较低，当某些节点处于空闲状态时，可以将工作负载过大的结点上的部分任务通过高速网传送给空闲结点处理，从而实现系统的负载平衡。
但是在分布式数据库系统中，为了适应应用的需要，满足部门分布特点的需要，各结点间一般采用局域网或广域网相连，网络带宽较低，点到点的通信开销较大。因此，在查询处理时一般应尽量减少结点间的数据传输量。

各节点的地位不同

在并行数据库系统中，各节点是完全非独立的，不存在全局应用和局部应用的概念，在数据处理中只能发挥协同作用，而不能有局部应用。在分布式数据库系统中，各结点除了能通过网络协同完成全局事务外，各节点具有场地自治性，每个节点是独立的数据库系统。每个节点有自己的数据库、客户、CPU等资源，运行自己的数据库管理系统，执行局部应用，具有高度的自治性。
