leetcode-283-movezeros

题目

Given an array nums, write a function to move all 0’s to the end of it while maintaining the relative order of the non-zero elements.

For example, given nums = [0, 1, 0, 3, 12], after calling your function, nums should be [1, 3, 12, 0, 0].

Note:
You must do this in-place without making a copy of the array.
Minimize the total number of operations.

解法1

解法2

分析

本题不难，分析后，我们要做的就是将一个数组中的零元素放到数组后面，而将非零元素放在数组前面，并不改变原数组中非零元素
的相对位置。

第一种方法，相对简单一点，就是按顺序遍历整个数组，碰到非零元素就将非零元素放到原数组中，非零元素先不管，被覆盖也无所
谓，并用标识符flag记录非零元素的数量，将所有数组元素遍历一遍后，便把所有的非零元素置于数组前面了，然后从数组中下标为flag的位开始为数组赋值 0，直到将数组填满。

第二种方法，代码量相对较多，单比较容易理解，思想就是遍历数组，将所有的非零元素放到另一个数组暂存，然后再将暂存非零元
素的数组的值逐一赋给原数组，再将原数组的剩余位用零填满。

制作yum源

背景

先说下为什么要制作本地yum源，在实际工程中，很多国企或者公司的某些研发环境为了网络安全，避免网络攻击，不会连接外网，在这种情况下就需要离线部署环境，此时就需要本地yum源和pip源，本篇仅介绍本地yum源制作方法，下篇介绍pip源制作方法。

制作本地yum源有三种方法：

一、
直接使用CentOS安装镜像文件作为本地yum源，优点是简单便捷，缺点是镜像文件中的软件包不完整，(centos 7 Everything才7G
左右)；
二、
同步CentOS官方源到本地（相当于创建了一个公共镜像），优点是软件包很全，缺点是占空间费流量；
三、
创建完全自定义的本地源，优点是灵活性最大，缺点是只是作为其他源的补充。

上面的三种方法，我自己试过第一种和第三种，第一种用centos 7 Everything的iso文件做本地yum源，安装scrapy时需要的依赖包
有些本地源中有，但有些并没有，因此第一种相对来说不太可靠。第三种很方便，需要什么软件就在能上网的电脑上提前下载好单不
安装，单独放在一个文件夹，把这个文件夹当作本地yum源。

通常，要使用某个类的方法，需要先实例化一个对象，再用对象调用方法。
而使用 @staticmethod或 @classmethod，则可以不需要实例化对象，直接用 类名.方法名() 的方式来调用。
这可以在一定程度上增加编写代码的效率，并可以使代码更加简洁。

既然 @staticmethod和 @classmethod都可以直接 用类名.方法名()的方式来调用，那他们有什么区别呢？
下面说一下它们的区别，从使用上来说：

如果在 @staticmethod 中要调用到这个类的一些属性或方法，只能直接类名.属性名或类名.方法名。
而 @classmethod 因为有cls参数，可以用来调用类的属性和方法或实例化对象等，避免硬编码。
